# ค่ายเยาวชน...รักษ์พงไพร ไฮเทค — ระบบให้คะแนนด้วย QR Code

เว็บแอประบบตั้งกลุ่ม / สแกน QR ให้คะแนนที่ฐาน / แดชบอร์ดสรุปผล
โฮสต์ฟรีด้วย **GitHub Pages** (หน้าเว็บ) + **Google Sheets + Apps Script** (ฐานข้อมูล/API)

## โครงสร้างไฟล์

```
rakpongprai-camp/
├── apps-script/
│   └── Code.gs          ← โค้ด Backend เอาไปวางใน Google Apps Script
└── docs/                 ← โฟลเดอร์นี้ทั้งหมดเอาขึ้น GitHub Pages
    ├── config.js          ← ใส่ URL ของ Apps Script ที่นี่ (จุดเดียว)
    ├── styles.css
    ├── index.html         ← หน้าตั้งกลุ่ม + สร้าง QR
    ├── scan.html          ← หน้าครูประจำฐาน สแกน + ให้คะแนน
    └── dashboard.html      ← หน้าแดชบอร์ดรวม
```

## ขั้นตอนที่ 1 — ตั้งค่า Google Sheets + Apps Script

1. ไปที่ [sheets.google.com](https://sheets.google.com) สร้างไฟล์ Sheet ใหม่ ตั้งชื่อ เช่น "ค่ายรักษ์พงไพร-ฐานข้อมูล"
2. เมนู **Extensions > Apps Script**
3. ลบโค้ดเดิมทั้งหมด แล้ววางโค้ดจากไฟล์ `apps-script/Code.gs` ทับลงไป
4. กด **Save** (ไอคอนแผ่นดิสก์)
5. กด **Deploy > New deployment**
   - คลิกไอคอนเฟือง ⚙️ เลือกประเภท **Web app**
   - Description: ใส่อะไรก็ได้ เช่น "v1"
   - Execute as: **Me**
   - Who has access: **Anyone**
   - กด **Deploy**
6. ระบบจะขอสิทธิ์ (Authorize access) → เลือกบัญชี Google ของครู → กด **Advanced** > **Go to (ชื่อโปรเจกต์) (unsafe)** > **Allow**
   (ขึ้นแบบนี้เพราะเป็นสคริปต์ที่ยังไม่ได้ verify จาก Google ซึ่งปกติสำหรับสคริปต์ที่เราเขียนเอง ปลอดภัย)
7. คัดลอก **Web app URL** ที่ได้ (ลงท้ายด้วย `/exec`) เก็บไว้ก่อน
8. กลับไปที่ Google Sheets จะเห็นว่ามีแท็บ **Groups** และ **Scores** ถูกสร้างขึ้นอัตโนมัติเมื่อมีการเรียกใช้งานครั้งแรก (ถ้ายังไม่เห็น ลองรีเฟรชหน้า Sheets)

> ⚠️ ทุกครั้งที่แก้โค้ดใน Apps Script ภายหลัง ต้องกด **Deploy > Manage deployments** > กดไอคอนดินสอ > Version เลือก **New version** > **Deploy** ใหม่ทุกครั้ง ไม่งั้นเว็บจะยังเรียกโค้ดเวอร์ชันเก่าอยู่

## ขั้นตอนที่ 2 — ใส่ URL ลงในเว็บ

เปิดไฟล์ `docs/config.js` แก้บรรทัดนี้ให้เป็น URL ที่คัดลอกมาจากขั้นตอนที่ 1:

```js
const API_URL = 'https://script.google.com/macros/s/xxxxxxxxxxxxx/exec';
```

## ขั้นตอนที่ 3 — อัปโหลดขึ้น GitHub Pages

1. สร้าง repository ใหม่บน GitHub (public หรือ private ก็ได้ ถ้า private ต้องมี GitHub Pro ถึงจะเปิด Pages ได้)
2. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์ `docs/` ขึ้น repo (จะเก็บไว้ใน root หรือในโฟลเดอร์ชื่อ `docs` ก็ได้ ขอให้สอดคล้องกับการตั้งค่าขั้นตอนถัดไป)
3. ไปที่ repo > **Settings > Pages**
4. ในหัวข้อ **Build and deployment**
   - Source: **Deploy from a branch**
   - Branch: เลือก branch หลัก (เช่น `main`) และเลือกโฟลเดอร์ **/docs** (หรือ **/(root)** ถ้าอัปโหลดไฟล์ไว้ที่ root)
5. กด **Save** รอสัก 1-2 นาที จะได้ลิงก์เว็บ เช่น `https://ชื่อผู้ใช้.github.io/rakpongprai-camp/`

## ขั้นตอนที่ 4 — วิธีให้คะแนนหน้างานจริง (อัปเดตล่าสุด)

QR ของแต่ละกลุ่มตอนนี้เก็บเป็น **ลิงก์เต็ม** (ไม่ใช่แค่รหัสกลุ่มเปล่าๆ) ครูจึงใช้ **กล้องมือถือทั่วไป** (แอปกล้องเดิมของเครื่อง ไม่ต้องเปิดเว็บนี้ก่อน) ส่องที่ QR แล้วแตะลิงก์ที่ขึ้นมา จะเด้งเข้าหน้า `scan.html` พร้อมข้อมูลกลุ่มให้อัตโนมัติ

แต่ละฐานมี **รหัสประจำฐาน (station code)** คงที่ 1 รหัส (พิมพ์ป้ายจากหน้า `index.html` ส่วนล่างสุด ติดไว้ที่ฐาน):

| ฐาน | รหัสประจำฐาน |
|---|---|
| ฐาน 1: กู้ภัยพงไพร ⛰️ | 1401 |
| ฐาน 2: รวมพลังพิทักษ์ป่า 🛡️ | 1402 |
| ฐาน 3: ส่องสัตว์คู่ป่า 🐯 | 1403 |
| ฐาน 4: ลุยป่า ฝ่าดงระเบิด 🤝❤️ | 1404 |
| ฐาน 5: แยกขยะ สรรค์สร้างโลก 🌍✨ | 1405 |

**ขั้นตอนของครูที่หน้างาน:**

1. สแกน QR ของกลุ่ม → ระบบโชว์ชื่อกลุ่มและรายชื่อสมาชิกอัตโนมัติ
2. **ขั้นที่ 1** — พิมพ์รหัสประจำฐาน (ดูจากป้ายที่ติดอยู่ที่ฐาน) → ต้องพิมพ์ทุกครั้งที่สแกนกลุ่มใหม่ เพื่อยืนยันว่าให้คะแนนถูกฐาน ระบบจะโชว์ชื่อฐานให้เช็คก่อน
3. **ขั้นที่ 2** — แตะให้ดาว 1-5 ดวง ใส่คำติชม (ถ้ามี) แล้วกดบันทึก

> 💡 ถ้ากล้องมือถือของครูบางเครื่องเปิดลิงก์ไม่ติด ในหน้า `scan.html` มีปุ่มสำรอง **"สแกนด้วยกล้องในหน้านี้แทน"** และช่องพิมพ์รหัสกลุ่มเองด้วย

### เช็กลิสต์ก่อนใช้งานจริง

- [ ] เปิดหน้า `index.html` → สร้างกลุ่มทดสอบ 1 กลุ่ม → เช็คว่า QR ขึ้นและมีแถวใหม่ใน Google Sheets แท็บ Groups
- [ ] พิมพ์ป้ายรหัสประจำฐาน (ส่วนล่างของหน้า `index.html`) ติดไว้ที่แต่ละฐาน
- [ ] ทดสอบสแกน QR ของกลุ่มทดสอบด้วย **กล้องมือถือจริง** (ไม่ใช่ในแอป) ว่าเปิดลิงก์ไปหน้า `scan.html` ได้
- [ ] พิมพ์รหัสฐาน เช่น `1403` → เช็คว่าระบบโชว์ชื่อฐานถูกต้อง → ให้ดาว → กดบันทึก → เช็คว่ามีแถวใหม่ในแท็บ Scores
- [ ] เปิดหน้า `dashboard.html` → เช็คว่าเห็นกลุ่มทดสอบและดาวที่เพิ่งให้
- [ ] ทดสอบว่าเมื่อ **สแกนกลุ่มเดิม ฐานเดิมซ้ำ** ระบบจะ "แก้ไขคะแนนเดิม" แทนการบวกซ้ำ (ตามที่ตั้งใจออกแบบไว้)
- [ ] ทดสอบสัญญาณ WiFi/เน็ตมือถือบริเวณที่จะใช้จริง เพราะทุกฐานต้องต่อเน็ตเพื่อคุยกับ Google Sheets

## หมายเหตุสำคัญ

- **ข้อมูลเป็นแบบ real-time เกือบสด**: หน้าแดชบอร์ดจะดึงข้อมูลใหม่ทุก 3 วินาที ไม่ใช่ push ทันทีเป๊ะๆ แต่เนียนพอสำหรับงานค่าย
- **การกันสแกนซ้ำนับคะแนนซ้ำ**: ระบบผูกคะแนนกับ (กลุ่ม + ฐาน) คู่เดียว ถ้าครูสแกนกลุ่มเดิมที่ฐานเดิมอีกครั้งจะเป็นการ "แก้ไขคะแนนเดิม" ไม่ใช่บวกเพิ่ม
- **โควตา Google Apps Script**: บัญชี Gmail ฟรีเรียก API ได้ประมาณ 20,000 ครั้ง/วัน ซึ่งเพียงพอมากสำหรับค่ายขนาดหลายสิบ-หลายร้อยกลุ่ม
- **ไอคอนประจำฐาน/ชื่อฐาน/รหัสประจำฐาน** กำหนดไว้ 2 ที่ที่ต้องตรงกัน: `apps-script/Code.gs` ตัวแปร `BASES` และ `docs/config.js` ตัวแปร `BASES_CLIENT` — ถ้าจะแก้ชื่อฐานหรือรหัส ต้องแก้ทั้งสองที่แล้ว deploy Apps Script ใหม่
